Hold a key. Speak. The words land where you were already typing.

whisprd is a small daemon for Linux. Press and hold your hotkey, talk, let go — the transcript is typed into whatever window has focus. No window to switch to, nothing to paste.

the signal path

Four steps, and none of them is a window you have to look at.

The daemon holds no model and does no inference. It captures audio, posts it to OpenAI, and types back the answer. That is the whole program.

01

evdev watches one key

An epoll loop reads /dev/input/event*. The input thread never blocks — it flips an atomic and goes straight back to waiting, so release is instant however slow the network is.

02

Capture runs the whole time

The stream is open at 16 kHz mono S16 — exactly what Whisper wants, so nothing is ever resampled. A 250 ms rolling pre-roll is glued to the front of each utterance, which is how the first consonant survives the gap between your finger and the daemon noticing.

03

One multipart POST

The audio becomes a WAV in memory and goes to /audio/transcriptions. The queue is bounded and drops the oldest, so a slow endpoint can stall a transcript but never the microphone.

04

Keystrokes, not a clipboard

Where the compositor allows it, whisprd types the text as real key events. The virtual keyboard carries its own keymap, so your layout does not matter and no modifier bookkeeping is needed.

read this first

Everything you dictate is uploaded to OpenAI.

That is the design, not a footnote. whisprd is a thin client, not a local transcriber: you bring an API key and every utterance becomes a paid API call. Do not speak anything into it you would not paste into a web form.

Pointing it at a local OpenAI-compatible server works — the request is identical either way — but that path is untested and unsupported.

three backends

Typing into a Wayland window is three different problems, so there are three answers.

Auto-detection takes the first one that probes clean and prefers real typing. Override it with backend = in the config.

wlr-vk

hyprland · sway · river

Uploads its own keymap where every character it needs sits alone on its own key, so your active layout is irrelevant.

cost: the manager global exists only on wlroots
uinput

gnome · kde

Injects raw keycodes below the compositor, which then reads them through your layout. libxkbcommon works out which keycode and level makes each character, and the two cancel out.

cost: layout must match your active one
clipboard

anywhere · x11 too

Sets the clipboard and sends a paste chord. Layout-agnostic and works on everything. The fallback that always lands.

cost: your clipboard is overwritten
the panel

Pick a microphone by watching a bar move, not by trusting a device name.

whisprd-menu is the settings and history panel — GJS and GTK4, sharing nothing with the daemon but a config file, a SIGHUP and the journal. The daemon keeps no GUI dependencies and still runs headless.

It carries a live level meter on the selected source, because a wrong source is the single biggest cause of nonsense transcripts. Whisper does not answer silence with an empty string; it answers with caption boilerplate memorised from its training data — Thank you., Subtitles by the Amara.org community, a stray copyright line. whisprd refuses to send anything peaking below 2 % of full scale and tells you the level it measured.

With history = on it also keeps past sessions, in 0600 files inside a 0700 directory. It is off by default — a verbatim record of everything said at a machine should be something you switch on, not something you find later.

get it running

Clone it and run the installer.

You need an OpenAI API key first — whisprd will not start without one. Fedora, Debian/Ubuntu and Arch are handled; anything else falls through with a warning and lets the build name what is missing.

shell
$ git clone https://github.com/ajr-khll/linux-transcription
$ cd linux-transcription
$ ./install.sh

It installs dependencies, builds, adds you to the input group, seeds the config, asks for your key and enables the systemd user service. Log out and back in afterwards — group membership is read when your session starts, so until then the service cannot open /dev/uinput.

It is deliberately not a curl | bash one-liner. This needs sudo, and piping an unread script into a shell that then asks for your password is a habit worth not teaching.

then
$ whisprd --list-sources   # samples each device, prints its peak level
$ whisprd --say "hello"    # test injection without speaking
$ whisprd-menu             # settings and history

Do not run it as root. It reads /dev/input/event* and writes /dev/uinput; on most distributions both are root:input, so joining the group is enough.

What is built, and what is not.

This is pre-release. TODO.md in the repo carries the full list, including the gaps that matter before anyone calls it 1.0.

Known gaps worth naming: the uinput backend drops characters needing dead-key or compose sequences, the clipboard backend does not restore what it overwrote, and one paste chord applies to every app — so a config tuned for terminals behaves oddly everywhere else.
